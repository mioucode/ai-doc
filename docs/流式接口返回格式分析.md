# 流式接口返回格式分析文档

> 基于测试用例 `测试用例-2026-05-19.xlsx` 及现有代码实现分析得出

## 1. 接口概述

### 1.1 基本信息

| 项目 | 说明 |
|------|------|
| 接口路径 | `GET /api/agentloop/conversations/{conversationId}/events?run_id={runId}` |
| 协议 | SSE (Server-Sent Events) |
| Content-Type | `text/event-stream` |
| 请求头 | `X-Agent-Id: {agentId}` |
| 触发时机 | 调用 `POST /api/agentloop/conversations/{conversationId}/run` 后获取 `streamUrl` |

### 1.2 请求示例

```http
GET /api/agentloop/conversations/59e92e6f-5271-4866-be89-932af73f40ff/events?run_id=59e92e6f-5271-4866-be89-932af73f40ff
Accept: text/event-stream
X-Agent-Id: o7VWjb
```

## 2. SSE 事件流结构

### 2.1 事件顺序

一条完整的 SSE 流按以下顺序推送事件：

```
1. message_start                    → 流开始
2. content_block_start              → 内容块开始（规划/检索/写作）
3. content_block_delta (可选)       → 增量文本（仅写作阶段持续推送）
4. content_block_stop               → 内容块结束，附带阶段产物
5. message_stop                     → 流结束
6. data: [DONE]                     → 终止哨兵（非 JSON）
```

### 2.2 事件格式

每条事件以 `data:` 开头，事件之间以 `\n\n` 分隔：

```
data: {"type":"message_start"}

data: {"type":"content_block_start","content_block":{"type":"tool_use","skillName":"a2a_planning","displayText":"进行中：任务规划"}}

data: {"type":"content_block_delta","delta":{"type":"text_delta","text":"这是增量文本..."}}

data: {"type":"content_block_stop","payload":{"tool":"a2a_planning","plan":{"intent":"...","summary":"...","steps":[]}}}

data: {"type":"message_stop"}

data: [DONE]
```

## 3. 事件类型详解

### 3.1 message_start

流开始的标志，无额外数据。

```json
{
  "type": "message_start"
}
```

### 3.2 content_block_start

内容块开始，根据 `type` + `skillName` + `purpose` 区分阶段。

#### 规划阶段 (a2a_planning)

```json
{
  "type": "content_block_start",
  "content_block": {
    "type": "tool_use",
    "skillName": "a2a_planning",
    "displayText": "进行中：任务规划"
  }
}
```

#### 检索阶段 (retrieval)

```json
{
  "type": "content_block_start",
  "content_block": {
    "type": "tool_use",
    "skillName": "retrieval",
    "displayText": "进行中：资料检索"
  }
}
```

#### 写作阶段 (writing)

```json
{
  "type": "content_block_start",
  "content_block": {
    "type": "text",
    "purpose": "article_draft",
    "skillName": "writing",
    "displayText": "进行中：公文写作"
  }
}
```

#### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | `tool_use` 或 `text` |
| skillName | string | `a2a_planning` / `retrieval` / `writing` / `general` / `main_agent` |
| purpose | string | 文本用途，写作阶段为 `article_draft` |
| displayText | string | UI 展示文案，如"进行中：XXX" |

### 3.3 content_block_delta

增量文本推送，**仅写作阶段**会持续推送。

```json
{
  "type": "content_block_delta",
  "delta": {
    "type": "text_delta",
    "text": "这是一段增量文本内容..."
  }
}
```

#### 思考流（简单回答场景）

```json
{
  "type": "content_block_delta",
  "delta": {
    "type": "thinking_delta",
    "thinking": "思考过程内容..."
  }
}
```

#### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | `text_delta` 或 `thinking_delta` |
| text | string | 增量文本内容 |
| thinking | string | 思考流内容 |

### 3.4 content_block_stop

内容块结束，**阶段产物全部承载在 `payload` 字段**。

#### 规划阶段产物

```json
{
  "type": "content_block_stop",
  "payload": {
    "tool": "a2a_planning",
    "skillName": "a2a_planning",
    "displayText": "已完成：任务规划",
    "sourceState": "model_success",
    "errorDetail": null,
    "steps": 3,
    "plan": {
      "intent": "用户意图描述",
      "summary": "规划摘要",
      "steps": [
        {
          "index": 1,
          "skillName": "retrieval",
          "title": "资料检索",
          "displayTitle": "检索相关资料",
          "dependsOn": [],
          "subtaskRole": "skill_worker"
        },
        {
          "index": 2,
          "skillName": "writing",
          "title": "公文写作",
          "displayTitle": "撰写公文正文",
          "dependsOn": ["retrieval-1"],
          "subtaskRole": "skill_worker"
        }
      ]
    }
  }
}
```

#### 检索阶段产物

```json
{
  "type": "content_block_stop",
  "payload": {
    "tool": "retrieval",
    "skillName": "retrieval",
    "displayText": "已完成：资料检索",
    "sourceState": "model_success",
    "errorDetail": null,
    "normalizedResult": {
      "source": "search_engine",
      "items": [
        {
          "title": "文章标题1",
          "description": "文章摘要描述...",
          "url": "https://example.com/article1"
        },
        {
          "title": "文章标题2",
          "description": "文章摘要描述...",
          "url": "https://example.com/article2"
        }
      ],
      "itemsTotal": 2
    }
  }
}
```

#### 写作阶段产物

```json
{
  "type": "content_block_stop",
  "payload": {
    "tool": "writing",
    "skillName": "writing",
    "displayText": "已完成：公文写作",
    "sourceState": "model_success",
    "errorDetail": null,
    "normalizedResult": {
      "document": "<h1>公文标题</h1><p>公文正文内容...</p>"
    }
  }
}
```

#### 主 Agent 直答产物

```json
{
  "type": "content_block_stop",
  "payload": {
    "tool": "main_agent",
    "displayText": "已完成：回答",
    "normalizedResult": {
      "text": "这是主 Agent 的直接回答文本..."
    }
  }
}
```

#### payload 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| tool | string | `a2a_planning` / `retrieval` / `writing` / `main_agent` |
| skillName | string | 技能名称 |
| displayText | string | 完成态文案，如"已完成：XXX" |
| sourceState | string | `legacy_success` / `model_success` |
| errorDetail | any | 错误详情，正常完成时为 null |
| steps | number | 规划阶段的步骤数量 |
| plan | object | 规划产物，包含 intent/summary/steps |
| normalizedResult | object | 检索/写作阶段产物 |
| normalizedResult.items | array | 检索结果条目 |
| normalizedResult.document | string | 写作最终文档 HTML |
| normalizedResult.text | string | 主 Agent 直答文本 |

### 3.5 message_stop

流结束标志。

```json
{
  "type": "message_stop"
}
```

### 3.6 终止哨兵

```
data: [DONE]
```

**注意**：这不是 JSON，需要显式跳过。

## 4. 阶段状态机

```
                    start(a2a_planning)              stop(a2a_planning)
                null ───────────────────► planning ──────────────────► null

                    start(retrieval)                 stop(retrieval)
                null ───────────────────► retrieval ─────────────────► null

                    start(text/article_draft)        stop(writing)
                null ───────────────────► writing ──────────────────► null
```

### 4.1 各阶段对应关系

| 阶段 | content_block_start 特征 | delta | content_block_stop 产物 |
|------|-------------------------|-------|------------------------|
| 规划 | `type=tool_use`, `skillName=a2a_planning` | 无 | `payload.plan` |
| 检索 | `type=tool_use`, `skillName=retrieval` | 无 | `payload.normalizedResult.items` |
| 写作 | `type=text`, `purpose=article_draft`, `skillName=writing` | `delta.text` | `payload.normalizedResult.document` |

## 6. TypeScript 类型定义

```typescript
/** 工具调用块：用于规划/检索等非纯文本阶段 */
export interface ToolUseBlock {
  type: 'tool_use'
  skillName: 'a2a_planning' | 'retrieval' | string
  displayText?: string
}

/** 纯文本块：写作阶段使用 */
export interface TextBlock {
  type: 'text'
  purpose?: 'article_draft' | string
  skillName?: 'writing' | string
  displayText?: string
}

export type ContentBlock = ToolUseBlock | TextBlock

/** 写作阶段每次推送的增量文本 */
export interface TextDelta {
  type: 'text_delta'
  text: string
}

/** 思考流 */
export interface ThinkingDelta {
  type: 'thinking_delta'
  thinking: string
}

/** 规划步骤 */
export interface PlanStep {
  index: number
  skillName: string
  title: string
  displayTitle?: string
  dependsOn?: string[]
  subtaskRole?: string
}

/** content_block_stop 的 payload */
export interface ToolStopPayload {
  tool: 'a2a_planning' | 'retrieval' | 'writing' | 'main_agent' | string
  skillName?: string
  stepIndex?: number
  stepTitle?: string
  displayText?: string
  retryable?: boolean
  sourceState?: 'legacy_success' | 'model_success' | string
  errorDetail?: unknown
  steps?: number
  plan?: {
    intent?: string
    summary?: string
    steps: PlanStep[]
  }
  normalizedResult?: {
    source?: string
    items?: Array<{ title?: string; description?: string; url?: string }>
    itemsTotal?: number
    document?: string
    text?: string
  }
}

/** 基础事件 */
export interface StreamEventBase {
  type: string
  [key: string]: unknown
}

/** 各事件类型 */
export interface MessageStartEvent extends StreamEventBase {
  type: 'message_start'
}

export interface ContentBlockStartEvent extends StreamEventBase {
  type: 'content_block_start'
  content_block?: ContentBlock
}

export interface ContentBlockDeltaEvent extends StreamEventBase {
  type: 'content_block_delta'
  delta?: TextDelta | ThinkingDelta | (Record<string, unknown> & { text?: string; thinking?: string })
}

export interface ContentBlockStopEvent extends StreamEventBase {
  type: 'content_block_stop'
  payload?: ToolStopPayload
}

export interface MessageDeltaEvent extends StreamEventBase {
  type: 'message_delta'
  delta?: { stop_reason?: string }
}

export interface MessageStopEvent extends StreamEventBase {
  type: 'message_stop'
}

export type StreamEvent =
  | MessageStartEvent
  | ContentBlockStartEvent
  | ContentBlockDeltaEvent
  | ContentBlockStopEvent
  | MessageDeltaEvent
  | MessageStopEvent
  | StreamEventBase
```

## 7. 前端对接要点

### 7.1 SSE 连接建立

```typescript
const controller = new AbortController()

fetch(streamUrl, {
  credentials: 'include',
  signal: controller.signal,
  headers: {
    Accept: 'text/event-stream',
    'X-Agent-Id': agentId,
  },
})
```

### 7.2 事件解析约定

1. 每条事件以两个换行 `\n\n` 分隔
2. 每行以 `data:` 开头，data 内容直接 JSON.parse
3. `data: [DONE]` 是流终止哨兵，非 JSON，必须显式跳过
4. 事件类型可能是 kebab-case，需统一转换为下划线格式

### 7.3 事件处理流程

```typescript
// 伪代码示例
function handleEvent(event: StreamEvent) {
  switch (event.type) {
    case 'message_start':
      // 初始化消息结构
      break
    case 'content_block_start':
      // 根据 block.type + skillName 判断阶段
      // 创建对应的 step 占位
      break
    case 'content_block_delta':
      // 写作阶段：增量文本写入 document body
      // 思考流：可忽略或展示
      break
    case 'content_block_stop':
      // 根据 payload.tool 处理阶段产物
      // planning: 解析 plan.steps 创建任务进度卡
      // retrieval: 渲染检索结果
      // writing: 用 normalizedResult.document 覆盖累计 delta
      // main_agent: 渲染纯文本回答
      break
    case 'message_stop':
      // 兜底：将所有残留任务置 completed
      break
  }
}
```

### 7.4 注意事项

1. **规划步骤数量 ≤ 1 时**：不展示规划相关卡片，直接进入任务执行
2. **写作阶段**：delta 持续推送，stop 时用 `normalizedResult.document` 覆盖最终内容
3. **思考流**：`type=thinking` 的 content_block 不展示、不落步骤
4. **事件类型大小写**：后端可能发送 kebab-case，前端需统一转换
5. **异常处理**：非 JSON 片段需静默忽略

## 8. 错误处理

### 8.1 错误状态标识

```json
{
  "type": "content_block_stop",
  "payload": {
    "tool": "retrieval",
    "displayText": "已完成：资料检索",
    "sourceState": "legacy_success",
    "errorDetail": {
      "message": "检索失败原因",
      "code": "RETRIEVAL_ERROR"
    }
  }
}
```

### 8.2 可重试标识

```json
{
  "payload": {
    "tool": "writing",
    "retryable": true,
    "errorDetail": { "message": "写作中断，可重试" }
  }
}
```
